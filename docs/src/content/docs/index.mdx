<section class="content-section faq-section">
  <div class="section-badge">FAQ</div>
  <h2>Frequently Asked Questions</h2>

  <div class="faq-grid">
    <details class="faq-item">
      <summary>
        <span class="faq-question">Why not just use sub-agents in Claude Code or Cursor?</span>
        <span class="faq-toggle"></span>
      </summary>
      <div class="faq-answer">
        Sub-agents are session-scoped ‚Äî they lose context the moment your session ends. Neova gives your agents persistent, structured context that lives across sessions. Plus, it's agent-agnostic: it works with any coding agent through its MCP server or by simply reading files from your repo. No lock-in to a single tool.
      </div>
    </details>

    <details class="faq-item">
      <summary>
        <span class="faq-question">How is this different from Jira, Linear, or other PM tools?</span>
        <span class="faq-toggle"></span>
      </summary>
      <div class="faq-answer">
        Traditional PM tools are designed for humans to read. Neova is designed for agents <em>and</em> humans to collaborate on. Tasks are living documents that both sides read from and write to ‚Äî creating a shared workspace where context builds up over time, not a ticket graveyard.
      </div>
    </details>

    <details class="faq-item">
      <summary>
        <span class="faq-question">Which coding agents does it work with?</span>
        <span class="faq-toggle"></span>
      </summary>
      <div class="faq-answer">
        All of them. Neova stores everything as plain files in your repository, so any agent that can read files can use it. For deeper integration, the built-in MCP server lets agents like Claude Code, Cursor, Windsurf, and others interact with your board directly.
      </div>
    </details>

    <details class="faq-item">
      <summary>
        <span class="faq-question">Do I need to change my existing workflow?</span>
        <span class="faq-toggle"></span>
      </summary>
      <div class="faq-answer">
        No. You define your own stages and templates ‚Äî Neova adapts to how you already work. Start with our defaults and customize as you learn what works best for your team.
      </div>
    </details>

    <details class="faq-item">
      <summary>
        <span class="faq-question">Is my data safe? Where is it stored?</span>
        <span class="faq-toggle"></span>
      </summary>
      <div class="faq-answer">
        Everything is stored locally as files in your repository. We don't store anything ‚Äî no cloud, no database, no telemetry. Your data never leaves your machine. You own it completely.
      </div>
    </details>

    <details class="faq-item">
      <summary>
        <span class="faq-question">Is it open source?</span>
        <span class="faq-toggle"></span>
      </summary>
      <div class="faq-answer">
        No, but it's completely free to use. We believe in giving developers great tools without barriers.
      </div>
    </details>

    <details class="faq-item">
      <summary>
        <span class="faq-question">Can I use it with a team?</span>
        <span class="faq-toggle"></span>
      </summary>
      <div class="faq-answer">
        Yes! Since Neova is git-native, it fits right into any team workflow. If your team collaborates through a shared repo, Neova works out of the box. Branch, merge, and review tasks just like code.
      </div>
    </details>

    <details class="faq-item">
      <summary>
        <span class="faq-question">How do I get started?</span>
        <span class="faq-toggle"></span>
      </summary>
      <div class="faq-answer">
        Download Neova, open your project folder, and you're ready to go. We provide starter templates so you can be up and running in minutes. Connect your agent via MCP for the full experience.
      </div>
    </details>
  </div>
</section>
